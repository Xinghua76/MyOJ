- 注册：邮箱注册；校验唯一性；发送验证码；校验验证码后创建用户。

1) 密码必须强散列（bcrypt/argon2）+ 加盐；验证码有过期与限流；防爆破（同IP/同账号限速）。
2) Token 设计：access + refresh（可选）；支持注销与强制失效。
3) 错误码与提示文案要适合前端展示。
输出：
- 需要新增/修改的表与 Redis key 设计
- 后端接口（注册/发码/登录/刷新/登出/重置密码）
- 核心代码（控制器/服务/中间件/过滤器）
- 单元测试与接口测试用例
